# 数据库表结构说明

数据库文件：`data/etf_manager.db`（SQLite，WAL模式），共 7 张表。

## 表命名规范

- **命名方式**：蛇形命名法（snake_case）
- **表名规则**：复数形式或名词形式，业务语义清晰
- **字段命名**：
  - 主键：表名单数_id（如 holding_id, plan_id）
  - 时间：*_at（如 created_at）或 *_date（如 first_buy_date）
  - 布尔：is_*（如 is_active）
  - 金额：amount 或 *_amount
  - 份额：shares 或 *_shares

## 1. fund_info — 基金基本信息表

| 字段 | 类型 | 说明 |
|------|------|------|
| fund_code | TEXT | 基金代码（主键），如 159770 |
| fund_name | TEXT | 基金名称，如 机器人ETF |
| fund_company | TEXT | 基金公司，如 易方达基金 |
| fund_type | TEXT | 基金类型，如 ETF |
| tracking_index | TEXT | 跟踪指数，如 中证机器人指数 |
| risk_level | TEXT | 风险等级，如 中高风险 |
| fund_category | TEXT | 自定义分类，如 军工/芯片/黄金/机器人/互联网 |
| top_holdings | TEXT | 前十大持仓（JSON或文本） |
| risk_points | TEXT | 风险提示要点 |
| return_1y | TEXT | 近1年收益率 |
| return_3y | TEXT | 近3年收益率 |
| return_since_inception | TEXT | 成立以来收益率 |
| created_at | TEXT | 创建时间，自动填充 |
| updated_at | TEXT | 更新时间，自动填充 |

## 2. fund_holdings — 持仓管理表

记录每支基金在不同平台的持仓情况，同一基金在不同平台各一条记录。

| 字段 | 类型 | 说明 |
|------|------|------|
| holding_id | INTEGER | 持仓记录ID（主键，自增） |
| fund_code | TEXT | 基金代码（外键 → fund_info） |
| fund_name | TEXT | 基金名称，如 机器人ETF |
| platform | TEXT | 持仓平台，如 支付宝/涨乐财富通 |
| holding_shares | REAL | 持有份额（总份数） |
| avg_buy_price | REAL | 平均买入价（持仓成本价）= 累计投入 ÷ 持有份额 |
| base_shares | REAL | 底仓份额（不可卖出的1/5仓位） |
| tradable_shares | REAL | 可交易份额 |
| current_price | REAL | 当前价格（净值，元/份或元/股） |
| holding_value | REAL | 持仓总值（市值）= 持有份额 × 当前价格 |
| invested_capital | REAL | 累计投入金额（本金总额，含手续费） |
| profit_loss_amount | REAL | 盈亏金额 = 持仓总值 - 累计投入金额 |
| return_rate | REAL | 收益率(%) = (持仓总值 - 累计投入金额) ÷ 累计投入金额 × 100 |
| first_buy_date | TEXT | 首次买入日期 |
| last_update_date | TEXT | 最后更新日期 |
| created_at | TEXT | 创建时间，自动填充 |
| updated_at | TEXT | 更新时间，自动填充 |

> **唯一约束**：(fund_code, platform)，同一基金在同一平台只有一条记录。

**计算公式**：
```
平均买入价 = 累计投入金额 ÷ 持有份额
持仓总值 = 持有份额 × 当前价格
盈亏金额 = 持仓总值 - 累计投入金额
收益率 = (持仓总值 - 累计投入金额) ÷ 累计投入金额 × 100%
```

## 3. daily_quotes — 每日净值表

存储每支基金的每日行情数据，用于生成收益曲线和触发交易信号。

| 字段 | 类型 | 说明 |
|------|------|------|
| quote_id | INTEGER | 记录ID（主键，自增） |
| fund_code | TEXT | 基金代码（外键 → fund_info） |
| fund_name | TEXT | 基金名称，如 机器人ETF |
| quote_date | TEXT | 净值日期，格式 YYYY-MM-DD |
| nav | REAL | 单位净值 |
| acc_nav | REAL | 累计净值 |
| daily_change_pct | REAL | 当日涨跌幅（%） |
| daily_value | REAL | 当日持仓市值 |
| daily_pnl | REAL | 当日盈亏金额 |
| created_at | TEXT | 创建时间，自动填充 |

> **唯一约束**：(fund_code, quote_date)，每支基金每天只有一条记录。
> **索引**：fund_code、quote_date 字段均已建索引，查询性能有保障。

---

## 4. dca_plans — 定投计划表

管理所有基金的定投计划，包括频率、金额、类型等。

| 字段 | 类型 | 说明 |
|------|------|------|
| plan_id | INTEGER | 计划ID（主键，自增） |
| fund_code | TEXT | 基金代码（外键 → fund_info） |
| fund_name | TEXT | 基金名称，如 机器人ETF |
| platform | TEXT | 定投平台 |
| is_active | INTEGER | 是否启用（1=启用，0=停用） |
| frequency | TEXT | 定投频率，如 每周/每两周/每月 |
| amount | REAL | 每期定投金额（元） |
| dca_type | TEXT | 定投类型，默认"固定金额"，也可为"智能定投" |
| total_invested | REAL | 该计划累计已投金额 |
| start_date | TEXT | 定投开始日期 |
| next_execute_date | TEXT | 下次执行日期 |
| created_at | TEXT | 创建时间，自动填充 |
| updated_at | TEXT | 更新时间，自动填充 |

> **索引**：fund_code、is_active、next_execute_date 字段均已建索引。

---

## 5. transactions — 交易记录表

记录每一笔实际发生的买入/卖出/定投交易。

| 字段 | 类型 | 说明 |
|------|------|------|
| tx_id | INTEGER | 交易ID（主键，自增） |
| fund_code | TEXT | 基金代码（外键 → fund_info） |
| fund_name | TEXT | 基金名称，如 机器人ETF |
| platform | TEXT | 交易平台 |
| tx_type | TEXT | 交易类型：买入/卖出/定投 |
| tx_date | TEXT | 交易日期 |
| amount | REAL | 交易金额（元） |
| shares | REAL | 交易份额 |
| nav_at_tx | REAL | 成交时净值 |
| fee | REAL | 手续费，默认0 |
| tx_status | TEXT | 交易状态：已成交/已撤销/待确认 |
| note | TEXT | 备注说明 |
| created_at | TEXT | 创建时间，自动填充 |

> **索引**：tx_date、fund_code、tx_type 字段均已建索引。

---

## 6. trading_rules — 交易规则配置表

将上方的交易规则结构化存储，用于自动生成交易信号。

| 字段 | 类型 | 说明 |
|------|------|------|
| rule_id | INTEGER | 规则ID（主键，自增） |
| fund_category | TEXT | 适用基金分类，如 军工/芯片/黄金 |
| rule_type | TEXT | 规则类型：止盈/接回/大跌加仓/核心纪律 |
| condition_desc | TEXT | 触发条件描述 |
| threshold | REAL | 阈值（如涨跌幅百分比） |
| action_desc | TEXT | 触发后的操作描述 |
| priority | INTEGER | 优先级（数字越大越优先） |
| is_active | INTEGER | 是否启用（1=启用，0=停用） |
| created_at | TEXT | 创建时间，自动填充 |
| updated_at | TEXT | 更新时间，自动填充 |

> **索引**：fund_category、rule_type、is_active 字段均已建索引。

---

## 7. trade_signals — 交易信号表

当满足交易规则条件时生成的待执行信号，支持状态跟踪。

| 字段 | 类型 | 说明 |
|------|------|------|
| signal_id | INTEGER | 信号ID（主键，自增） |
| fund_code | TEXT | 基金代码（外键 → fund_info） |
| fund_name | TEXT | 基金名称，如 机器人ETF |
| signal_date | TEXT | 信号生成日期 |
| signal_type | TEXT | 信号类型：止盈/接回/大跌加仓 |
| trigger_condition | TEXT | 具体触发条件描述 |
| trigger_value | REAL | 触发时的数值（如收益率、价格） |
| suggested_action | TEXT | 建议操作 |
| exec_status | TEXT | 执行状态：待执行/已执行/已忽略 |
| actual_action | TEXT | 实际执行的操作（手动填写） |
| note | TEXT | 备注说明 |
| created_at | TEXT | 信号创建时间，自动填充 |
| updated_at | TEXT | 更新时间，自动填充 |

> **索引**：signal_date、fund_code、exec_status 字段均已建索引。

## 表关系

```
fund_info (fund_code)
  ├── fund_holdings (fund_code) ── 1:N，一支基金可在多个平台持仓
  ├── daily_quotes (fund_code) ── 1:N，一支基金有多天净值
  ├── dca_plans (fund_code) ── 1:N，一支基金可有多个定投计划
  ├── transactions (fund_code) ── 1:N，一支基金有多笔交易
  └── trade_signals (fund_code) ── 1:N，一支基金可有多个信号

trading_rules ── 独立表，通过 fund_category 字段与 fund_info.fund_category 逻辑关联
```

## 优化改进说明

### 1. 表名统一化
- ✅ `my_holdings` → `fund_holdings`（去掉不规范的"my_"前缀）
- ✅ 与其他表命名保持一致

### 2. 字段命名规范化
- ✅ 主键统一：`id` → `quote_id`（daily_quotes表）
- ✅ 时间字段统一：`date` → `quote_date`（daily_quotes表）
- ✅ 新增通用字段：`created_at`、`updated_at`、`note`

### 3. 扩展性增强
- ✅ `fund_holdings`：新增 `last_update_date`（最后更新日期）
- ✅ `daily_quotes`：主键改为 `quote_id`
- ✅ `dca_plans`：新增 `next_execute_date`（下次执行日期）
- ✅ `transactions`：新增 `tx_status`（交易状态）
- ✅ `trade_signals`：新增 `trigger_value`（触发数值）、`note`（备注）
- ✅ 所有表：新增 `created_at`、`updated_at`（时间戳）

### 4. 数据一致性保障
- ✅ 所有业务表都有 `fund_name` 冗余字段（便于查询）
- ✅ 统一时间格式：`*_at`（时间戳）、`*_date`（日期）
- ✅ 统一状态字段命名：`is_active`、`exec_status`、`tx_status`

### 5. 性能优化
- ✅ 所有关键查询字段建立索引
- ✅ 合理设置唯一约束
